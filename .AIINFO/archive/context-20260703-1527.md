# CodexAppSwitcher 项目上下文

最近更新：2026-07-03 15:13 (Asia/Shanghai)

## 会话进度

已完成：

- 产品定位：Win11 本机 Codex App 多账号登录态切换器。
- 技术栈：C# + WPF + .NET 8 + WebView2。
- 登录态边界：只管理 `AppData\Roaming\Codex` 与可选 `.codex\auth.json`。
- 对话列表共享：不隔离 `.codex\sessions`、状态库、日志库和项目列表。
- 安全边界：不解密、不展示、不导出 token/cookie。
- UI 第一版已基本定稿：深色 dashboard、左侧导航、顶部工具栏、账号表格、右侧状态卡片和操作日志。
- logo 已改为从用户参考截图提取的透明 PNG，并完成渐变、铺满、比例调整。
- 已实现账号快照采集、元数据存储、文件锁检测、Codex App 运行检测、切换预检、真实切换执行器、rollback 恢复器和手动验证指引。
- 默认安全配置已恢复：真实切换关闭，宿主保护开启。
- 已新增账号用户标识字段，账号默认名称策略为用户 ID，例如邮箱；`.AIINFO` 不记录实际邮箱。
- 已新增 WebView2 添加账号窗口骨架：每个账号使用独立 WebView2 profile，打开 analytics 页面，用户登录后从页面可见文本提取邮箱样式用户 ID。
- “添加账号”按钮已接入 WebView2 识别流程：识别成功后保存 `DisplayName/UserIdentifier/BrowserProfilePath` 并采集当前 Codex App 快照；识别失败或取消时不保存账号。
- 添加账号窗口已补充兜底体验：支持“重新识别”页面可见账号 ID，也支持手动填写邮箱样式用户 ID；窗口 Owner 已绑定主窗口，避免脱离主窗口。
- 添加账号弹窗已改为深色自绘标题栏，左上角同步使用全局 `AppLogoIcon`，去除系统白色标题栏，并补充窗口拖动、最小化、最大化和关闭行为。
- 修复账号列表误显示“未命名账号”的原因：`AccountMetadataStore.LoadAll` 原先递归搜索所有 `metadata.json`，会把 WebView2 profile 缓存中的 `metadata.json` 当成账号；现已改为只读取 `accounts/{accountId}/metadata.json`。
- 根据用户实际登录截图修正账号识别：ChatGPT 登录后主界面不一定显示邮箱，识别逻辑已增强为扫描页面可见文本、`aria-label/title/alt/placeholder` 和按钮文本；优先识别邮箱，找不到邮箱时从账号区域提取显示名作为账号 ID 兜底。
- 账号识别主链路已改为受控 Profile API：在 WebView2 页面上下文内使用当前 profile 登录态 `fetch` 白名单账号接口，只提取 email/userId/name 等字段；页面文本识别降级为兜底，不读取 cookie/token/header，也不保存完整响应。
- 添加账号窗口已按用户要求拆分职责：`重新识别` 才会调用 Profile API / 页面邮箱兜底识别；`使用当前账号` 只检查已识别或手动填写的账号 ID。
- 添加账号窗口底部已新增醒目提示区；无账号 ID 时点击 `使用当前账号` 会提示重新识别或手动填写，有账号 ID 时会二次确认后才关闭弹窗。
- 添加账号窗口标题已统一为 `添加 ChatGPT 账号`，顶部重复标题已移除；`使用当前账号` 已改为强调按钮，取消/关闭在已有账号 ID 时会确认是否取消添加。
- 添加账号弹窗已移除 WebView 上方说明栏；登录后会轻量尝试打开左下角账号菜单并自动识别邮箱，以减少用户手动打开菜单和点击重新识别。
- 添加账号确认/取消提示已从系统 MessageBox 改为深色主题自定义弹窗；`使用当前账号` 按钮颜色已调暗以保证文字可读。
- 已修复 Google 登录页误触问题：自动打开账号菜单、Profile API 读取和页面邮箱扫描都限制为 `chatgpt.com` 域；Google 登录/注册页只等待用户完成登录。
- 已拦截 `support.google.com` 新窗口，避免 Google 账号帮助窗口被反复弹出。
- 主界面顶部按钮文案已从 `添加账号` 改为 `添加 ChatGPT 账号`。
- 当前账号识别优先级为：手动填写、自动或手动识别得到的 Profile API email/userId、页面文本中的邮箱；页面显示名如 `SetSail` 不会从页面文本自动获取。
- 已修复“使用当前账号后列表为空”：ChatGPT Web 账号确认成功后会先保存账号元数据和 WebView2 profile；Codex App 登录态快照采集失败只显示警告并标记账号为 `未接管`，不再阻止账号进入列表。
- 账号列表首列已增加深色主题悬停提示，展示账号、状态和说明；账号名与说明在列表内使用省略号截断，避免挤压后续列。
- `未接管` 状态已从绿色改为警告色，左侧状态点和状态徽标同步变色，避免误判为已接管。
- `刷新用量` 按钮已接入命令；`UsageStatusService` 已实现隐藏 WebView2 打开账号独立 profile 的 analytics 页面，并解析页面可见文本中的 5 小时、每周剩余百分比和重置时间。
- 用量刷新仍受 `EnableUsageRefresh` 控制，当前 `appsettings.json` 保持关闭；开关关闭时只写日志，不加载 analytics 页面。
- 刷新用量期间主按钮会显示不确定进度条和 `刷新中` 文案，并通过 ViewModel 状态拦截重复触发。
- 用量刷新成功后会保存到 `accounts/{accountId}/usage.json`；程序启动时会加载已有用量快照，避免重启后丢失上次刷新结果。
- 刷新用量按钮已增加 Tooltip，显示真实刷新开关状态；开关关闭时按钮透明度降低但仍可点击以写入明确日志说明。
- 账号列表用量展示已增加颜色阈值：剩余额度低于 25% 显示警告色，低于 10% 显示危险色，文本和进度条同步变化。
- 操作日志区域已改为默认滚动到最新记录，启动加载旧日志和新增日志都会自动定位到底部。
- 滚动条已改为全局深色主题样式：去除系统浅色箭头，轨道、滑块、悬停和拖动状态使用现有 dashboard 色板，并兼容水平与垂直滚动条。
- 账号表格已改为固定最小表格宽度并启用横向滚动，避免窗口缩小时账号名称列被其它固定列挤压到不可见。
- 用量重置时间已拆为 `5 小时重置` 与 `每周重置` 两列，行模型分别绑定 `FiveHourResetText` 与 `WeeklyResetText`。
- `5 小时重置` 列宽已从 142 收窄到 104，账号表格最小宽度同步从 1180 调整为 1142。
- 右侧回滚卡片已绑定最近 rollback 状态与“执行回滚”按钮；执行时先确认，再经过 Codex 进程关闭确认、真实切换开关和宿主保护拦截，默认安全配置下不会恢复真实登录态。
- 重复添加同一用户 ID 时已增加深色主题覆盖确认：确认后复用既有账号并用 staging/backup 流程覆盖 Web 登录态路径和 Codex App 快照；取消则不新增重复账号，默认仍不覆盖既有快照。
- 已新增无外部依赖的 `CodexAppSwitcher.Tests` 控制台测试项目；测试通过临时用户目录和临时数据根目录覆盖，验证切换准备会创建 rollback、Auth-only 替换 live `auth.json` 且不替换 Roaming、Auth+Roaming 会额外替换 Roaming。
- 切换边界测试已补充：缺失目标快照会拦截且不改当前登录态；宿主保护开启会拦截真实切换；Auth+Roaming 在 auth.json 被占用时会拦截。
- rollback 失败恢复测试已补充：通过锁定目标 auth 快照文件模拟应用目标登录态失败，验证切换失败后会从 rollback 恢复源账号 `auth.json`。
- 快照覆盖测试已补充：默认采集会拒绝覆盖既有快照并保持旧内容；传入 `allowOverwrite` 时会覆盖 Roaming 与 auth 快照，并清理 staging/backup 临时目录。
- 用量解析逻辑已暴露为可测试的纯解析方法；已补充中文 analytics 文本、英文文本、百分比越界 clamp、信息不完整不误报等边界测试，不启动 WebView2、不访问真实 ChatGPT。
- 用量刷新失败提示体验已优化：刷新完成后主界面最近操作会显示成功/总数和失败账号摘要，失败明细仍写入操作日志，避免部分失败时只显示成功数量。
- 手动验证指引已更新为当前功能版本，覆盖 UI 基础检查、添加 ChatGPT 账号、取消添加、重复账号覆盖、用量刷新、回滚卡片、Auth-only 与 Auth+Roaming（实验）真实切换边界和反馈模板。
- 刷新用量按钮悬浮提示已改为全局深色主题 Tooltip，文案收短并支持换行，避免系统白底低对比和提示过长。
- 已修复添加已登录账号后崩溃：重复账号覆盖确认依赖的 `PrimaryActionButtonStyle` 已提升到全局资源，避免主窗口确认框查找不到样式；账号快照采集中的 IO/权限异常也会转成失败结果，不再抛出导致程序退出。
- 用户已手动验证通过前四项：UI 基础显示、添加 ChatGPT 账号闭环、重复添加同一账号覆盖确认、刷新用量流程。
- 账号切换确认、关闭 Codex App 确认、执行回滚确认已从系统 MessageBox 改为深色主题弹框；宿主保护拦截逻辑保持不变。
- Codex App 关闭流程已从仅调用窗口关闭改为两段式：先请求主窗口正常关闭，超时后在真实切换开启且宿主保护关闭时结束残留 Codex 进程树；确认文案和手动验证指引已同步。
- 已定位切换未生效原因：本机 Codex App 是 WindowsApps 包应用，真实登录态目录在 `%LOCALAPPDATA%\Packages\OpenAI.Codex_*\LocalCache\Roaming\Codex`，不是旧的 `%APPDATA%\Codex`；路径解析已增加自动探测，找不到旧路径时自动使用包应用目录。
- 已新增账号行 `绑定` 操作：用于把当前本机 Codex App 登录态快照绑定到 Switcher 中已有账号，不再要求通过重复添加/覆盖账号来采集 App 快照；绑定前会二次确认目标账号，必要时走 Codex App 关闭确认。
- 手动验证指引已新增 `绑定当前 Codex App` 步骤：添加 ChatGPT 账号只负责 Web 登录态，`未接管` 账号通过 `绑定` 采集 App 快照。
- 绑定交互已补充加载态和防重复点击：绑定期间行按钮显示进度，已接管账号隐藏 `绑定` 按钮，仅保留 `切换`。
- Codex App 状态卡片已改为真实运行态绑定：构造、预检、关闭和绑定完成后刷新右上角徽标与安全检查状态，未运行时不再显示固定 `运行中`。
- 账号行操作已继续优化：当前使用账号隐藏 `切换` 按钮，未接管账号的 `绑定` 按钮加宽并显示 `绑定中` 加载文本，避免文字被挤压。
- 切换交互已补充加载态和防重复点击：未接管账号不显示 `切换`，已接管且非当前账号才可切换；切换期间显示 `切换中`，成功后同步当前账号元数据和右侧当前账号卡片。
- 未接管账号的行内操作已从 `绑定` 改为两段式 `接管`：第一次点击只进入接管流程，不把当前 Codex App 登录态写入目标账号；用户手动让 Codex App 登录目标账号后，再点击 `完成` 采集目标快照，避免误把当前已登录账号绑定给未接管账号。
- 针对用户反馈的“切换提示成功但 Codex App 未实际切换”已补防假成功机制：`AccountSwitchService` 在替换目标登录态后会比较目标 auth 快照与 live auth 文件指纹；Auth+Roaming（实验）还会校验 Roaming 附加状态，不一致时回滚并返回失败，不再标记当前账号。
- 切换流程已调整为确认后立即进入行内加载态，再关闭 App、替换、校验和启动，避免关闭/替换阶段界面先卡住再显示进度。
- `CodexProcessService` 已新增多入口启动 Codex App 的能力：优先使用 WindowsApps 包 AppID（从 `C:\Program Files\WindowsApps\OpenAI.Codex_*` 推断 `OpenAI.Codex_* !App`），再尝试 `codex:` 协议、开始菜单快捷方式和 `%LOCALAPPDATA%\OpenAI\Codex\bin\codex.exe`；顶部 `启动 Codex` 与右侧 `打开 Codex App` 已接入真实命令，启动成功必须检测到 Codex App 主窗口，只有后台进程不再判定为正常启动。
- 当前使用且已接管的账号行已新增 `更新` 按钮，用于在用户确认当前 Codex App 已登录该账号后覆盖更新该账号 App 快照；该按钮只对当前账号显示，避免误覆盖其它账号。
- 更新当前账号快照或完成接管时，如果工具曾为了采集快照关闭 Codex App，流程结束后会尝试恢复启动 Codex App；启动失败只写警告日志并保留原操作结果。
- 已接管但非当前账号行已新增 `重接管` 入口：点击后进入两段式流程，用户手动让 Codex App 登录目标账号，再点 `完成` 覆盖刷新目标账号快照，解决目标账号快照过旧或采错时切换不生效的问题。
- 切换执行成功/失败日志已增加诊断信息：实际登录态目录、目标快照文件数/大小、落地状态文件数/大小；安全检查也显示当前使用的 Roaming 登录态目录。
- 根据用户反馈，当前问题按切换执行失败处理，不再要求用户先重接管。已发现本机存在 Codex MSIX 包私有状态目录 `%LOCALAPPDATA%\Packages\OpenAI.Codex_2p2nqsd0c76g0`，普通权限访问返回 Access denied；如果账号状态部分存储在该目录，当前仅替换 `%APPDATA%\Codex` 属于不完整切换。
- `CodexPathResolver.CheckPackagedStateAccess` 已加入采集和切换预检：检测到包私有状态目录但无权限访问时，采集/切换会失败并提示“当前切换范围不完整”，避免继续出现替换部分目录后仍显示成功。
- 已新增独立提权 Helper 骨架项目 `CodexAppSwitcher.Elevated`，并加入解决方案；第一阶段只支持 `validate` + `DryRun=true`，用于校验一次性计划、白名单目录、目标路径和 reparse point，不执行任何真实复制、删除、替换或回滚。
- 提权 Helper 曾接入主程序切换入口的 `runas` dry-run gate；在后续改为 auth 主导且废弃 package-state 新切换后，该 gate 已退出主链路，`appsettings.json` 不再暴露该开关。
- 针对用户反馈“Codex App 打开仍是旧账号”，已确认 Roaming 文件一致不能证明真实账号切换成功；`CodexPathResolver` 已改为优先从 `%LOCALAPPDATA%\Packages\OpenAI.Codex_*` 发现包状态目录，并且只要发现 MSIX 包状态目录就阻断当前 Roaming-only 切换，避免再次报假成功。
- 参考 `xjoker/codex-switch` 后已补齐 CLI 运行态路径规则：`.codex\auth.json` 优先使用 `CODEX_HOME\auth.json`，否则回退用户目录 `.codex\auth.json`。

- 已通过主项目构建、测试项目构建、测试 DLL 执行和提权 Helper 构建验证；新增测试覆盖包状态快照采集与切换替换。

## 功能实现流程基线

1. 安全基线：真实切换默认关闭，宿主保护默认开启。
2. 账号添加闭环：WebView2 登录识别用户 ID，保存独立 profile，采集 App 快照。
3. 账号列表状态：显示用户 ID、快照状态、用量状态、当前使用状态。
4. 用量查询：按账号 profile 打开 analytics，解析 5 小时、周额度和重置时间；真实刷新受功能开关控制。
5. 切换预检：检查目标快照、进程、文件锁、rollback 和模式风险。
6. 真实切换：仅用户手动启动 Switcher 后验证，CLI/当前会话不执行。
7. rollback 恢复：显示最近 rollback 并支持一键恢复。
8. 验证收尾：补最小测试与手动验证文档。
9. 账号绑定：已有 Web 账号若未接管 App 快照，直接使用账号行 `绑定` 采集当前 Codex App 登录态。
10. 提权通道：先用独立 Helper 校验 plan/result 协议和路径白名单，再分阶段接入 `runas` 与真实文件操作。

## 重要决策

- 当前只保留 auth 主导测试切换：先关闭全部 Codex App，再替换 live `.codex\auth.json`；不替换 `%APPDATA%\Codex` 或 MSIX 包状态，不执行 Token 模块，不创建 rollback，不做落地校验。
- WebView2 profile 用于 ChatGPT 网页登录和 analytics 查询；Codex App 登录态快照是可选增强，采集失败不阻断 Web 账号保存。
- 账号用户 ID 可由登录后自动识别或用户点击 `重新识别` 触发，识别只在 `chatgpt.com` 域内通过 WebView2 Profile API 的 email/userId 和页面邮箱兜底；`使用当前账号` 不触发识别，不读取或输出 token/cookie。
- 真实关闭、真实替换登录态、真实切换不得在当前 Codex 会话内执行；提权 Helper 当前仅允许 dry-run 验证，不允许执行真实文件修改。
- 参考项目 `xjoker/codex-switch` 的关键可复用点是以 `auth.json` 作为账号切换单元，并支持 `CODEX_HOME` 覆盖；本项目只把 Roaming 作为实验附加状态，MSIX 包状态不再进入新接管/新切换。
